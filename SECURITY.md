# Security Policy

## Sicherheitsprobleme melden

Bitte melde Sicherheitsprobleme nicht mit vollständigen Exploit-Details in einem öffentlichen Issue.

Wenn GitHub Private Vulnerability Reporting für dieses Repository verfügbar ist, nutze bitte diesen Weg. Falls nicht, eröffne ein Issue mit einer knappen Beschreibung ohne sensible Details, damit ein privater Kontaktweg abgestimmt werden kann.

## Umfang

Dieses Projekt speichert keine Nutzereingaben dauerhaft und nutzt keine Datenbank. Relevant sind trotzdem zum Beispiel:

- unsichere Deployment-Konfigurationen
- versehentlich veröffentlichte Secrets
- Dependency-Schwachstellen
- Probleme, durch die Nutzereingaben geloggt oder offengelegt werden könnten

## Unterstützte Versionen

Aktuell wird nur der Stand auf `main` gepflegt.
